# 臨床心理学RAGナレッジベース（親子チャンク構造）

## 概要

SINR（Search is not Retrieval）手法に基づいた親子チャンク構造のRAGナレッジベースです。

## 構造

```
clinical_psych_rag_hierarchical/
├── parents/           # 親チャンク（LLM参照用）- 135個
│   ├── lecture01/
│   ├── lecture02/
│   └── ... lecture16/
├── children/          # 子チャンク（検索用）- 633個
│   ├── lecture01/
│   ├── lecture02/
│   └── ... lecture16/
├── mapping.json       # 親子関係マッピング
└── README.md
```

## チャンクサイズ

- **親チャンク**: 800-1200字（LLMへの入力用、文脈を保持）
- **子チャンク**: 200-400字（ベクトル検索用、精度重視）

## 使い方

### 検索時
1. ユーザーの質問を子チャンク（`children/`）でベクトル検索
2. ヒットした子チャンクの`parent_id`を取得
3. 対応する親チャンク（`parents/`）をLLMに渡す

### YAMLフロントマター

**親チャンク**:
```yaml
---
chunk_id: "01-01"
title: タイトル
keywords: [キーワード1, キーワード2]
children: ["01-01-c01", "01-01-c02", "01-01-c03"]
---
```

**子チャンク**:
```yaml
---
parent_id: "01-01"
child_id: "01-01-c01"
section: セクション名
keywords: [キーワード1, キーワード2]
char_count: 文字数
---
```

## mapping.json

親子関係のJSONマッピング：
```json
{
  "01-01": {
    "parent_file": "lecture01/01-01_xxx.md",
    "children": [
      {"child_id": "01-01-c01", "section": "...", "char_count": 250},
      ...
    ]
  }
}
```
